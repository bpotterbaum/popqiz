# Vercel Environment Variables Setup

## Required Environment Variables:

1. **NEXT_PUBLIC_SUPABASE_URL**
   - Your Supabase project URL
   - Example: `https://xxxxx.supabase.co`
   - Found in Supabase Dashboard > Settings > API

2. **NEXT_PUBLIC_SUPABASE_ANON_KEY**
   - Your Supabase anonymous/public key
   - Found in Supabase Dashboard > Settings > API

3. **SUPABASE_SERVICE_ROLE_KEY**
   - Your Supabase service role key (server-side only)
   - Found in Supabase Dashboard > Settings > API
   - ‚ö†Ô∏è Keep this secret! Never expose in client code

4. **OPENAI_API_KEY**
   - Your OpenAI API key
   - Found at https://platform.openai.com/api-keys
   - ‚ö†Ô∏è Keep this secret! Never expose in client code

---

## Option 1: Using Environment File (Recommended)

1. **Copy the example file:**
   ```bash
   cp .env.local.example .env.local
   ```

2. **Fill in your values in `.env.local`** (this file is gitignored for security)

3. **Add variables to Vercel using the script:**
   ```bash
   chmod +x vercel-env-setup.sh
   ./vercel-env-setup.sh
   ```

   Or manually add each one:
   ```bash
   npx vercel env add NEXT_PUBLIC_SUPABASE_URL production preview development
   # (paste your value when prompted)
   ```

---

## Option 2: Using Vercel Dashboard

1. Go to your Vercel project dashboard: https://vercel.com/dashboard
2. Select your project
3. Navigate to **Settings** > **Environment Variables**
4. Add each variable:
   - **Key**: The variable name (e.g., `NEXT_PUBLIC_SUPABASE_URL`)
   - **Value**: Your actual value
   - **Environments**: Select Production, Preview, and Development
5. Click **Save**
6. **Redeploy** your project after adding variables

---

## Option 3: Manual CLI Commands

```bash
# Login to Vercel first
npx vercel login

# Add each environment variable (you'll be prompted for the value)
npx vercel env add NEXT_PUBLIC_SUPABASE_URL production preview development
npx vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production preview development
npx vercel env add SUPABASE_SERVICE_ROLE_KEY production preview development
npx vercel env add OPENAI_API_KEY production preview development
```

---

## Verify Environment Variables

After adding variables, verify they're set:
```bash
npx vercel env ls
```
